# Release Checklist

This checklist keeps the public repository focused on the local-first code review tool and prevents local secrets or generated artifacts from being published.

## Before Pushing

- Run the demo readiness script:

```bash
scripts/check_demo_ready.sh
```

- Run the full test suite directly when changing core orchestration:

```bash
PYTHONPATH=src python3 -m unittest discover -s tests
```

- Run a syntax check:

```bash
PYTHONPYCACHEPREFIX=/private/tmp/macr_pycache PYTHONPATH=src python3 -m compileall -q src tests scripts
```

## Do Not Commit

- `.env` or any API key.
- `traces/*.json` generated by local runs.
- `.macr_cache/` code graph cache.
- `.macr_uploads/` uploaded zip extraction workspace.
- `patches/*.patch` generated locally.
- `external_data/` and `external_repos/`.
- Private planning notes excluded by `.gitignore`.

## Public Project Should Emphasize

- Local-first developer workflow.
- CLI and Web Review Workbench.
- Evidence-first code review using `rg`, AST, Symbol Graph, and Code Graph.
- Agent Board and Workflow Graph for structured multi-agent communication.
- Routing policy that skips unnecessary agents and LLM calls.
- Final Review, recovery, human-review escalation, and Monitor metrics.
- Offline evals, trace viewer, SARIF/GitHub PR outputs, and sample repo.

## Quick Verification Flow

```bash
python3 cli/agent_review.py ask \
  --repo sample_repos/sample_python_api \
  "这个接口在哪里鉴权？"

python3 cli/agent_review.py eval \
  --eval-file eval_sets/phase1.jsonl \
  --repo-root . \
  --report /tmp/macr_phase1_eval.md

python3 cli/agent_review.py view --port 8765
```

Open:

```text
http://127.0.0.1:8765
```

Only run this locally after creating `.env`; never commit the key.

```bash
set -a
source .env
set +a
python3 cli/agent_review.py llm-check --provider deepseek
```
